# Admin Guide

This guide helps administrators configure and manage Markdown Renderer for Confluence.

## Accessing the Admin Panel

1. Go to Confluence Administration
2. Navigate to Manage Apps
3. Find Markdown Renderer for Confluence
4. Click "Configure"

![Admin Panel](https://marketplace.atlassian.com/product-listing/files/f6817212-5bac-46f1-8d03-81850d18b316)

## Managing Templates

### Creating Templates

1. Go to the Templates section
2. Click "Create Template"
3. Enter a name and description
4. Write your template content
5. Save the template

### Template Best Practices

- Use clear, descriptive names
- Include placeholders for dynamic content
- Add helpful comments
- Keep templates focused and reusable
- Document template usage

## Permission Controls

### User Permissions

Configure who can:

- Create markdown content
- Edit existing content
- Use templates
- Access the admin panel

### Space Permissions

Set permissions at the space level:

- Allow/deny markdown usage
- Restrict template access
- Control attachment usage

## Security Settings

### Attachment Security

- Control attachment upload permissions
- Restrict file types
- Set size limits
- Configure storage locations

### Content Security

- Enable/disable external links
- Control macro usage
- Set content validation rules

## Best Practices

1. **Regular Review**: Periodically review permissions and templates
2. **Documentation**: Keep admin documentation up to date
3. **Backup**: Regularly backup templates and settings
4. **Testing**: Test changes in a staging environment
5. **Monitoring**: Monitor usage and performance

## Troubleshooting

Common admin issues and solutions:

1. **Permission Issues**

   - Verify user groups
   - Check space permissions
   - Review global settings

2. **Template Problems**

   - Validate template syntax
   - Check template permissions
   - Verify template variables

3. **Performance Issues**
   - Monitor system resources
   - Check attachment storage
   - Review caching settings
